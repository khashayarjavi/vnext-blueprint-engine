# Contributing

Thank you for contributing to vNEXT.

The goal of this repository is quality over speed.

## Before contributing

Every proposal should explain:

- Why the change exists
- What problem it solves
- Tradeoffs
- Failure modes
- Alternatives considered

## Preferred Contributions

- Decision logic
- Engineering research
- Standards mapping
- Blueprint quality improvements
- Architecture improvements
- Validation cases
- Red-team analysis
- Production testing

## Pull Requests

Large changes should begin with a discussion.

Every pull request should include:

- Motivation
- Design
- Risks
- Testing
